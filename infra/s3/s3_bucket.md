# Amazon S3 Configuration

## Overview

Amazon S3 is used in this project for storing processed flight data, reports, backups, and historical data generated by the flight tracking pipeline.

## AWS Managed Policy Used

Policy Name: AmazonS3FullAccess

Policy Type: AWS Managed Policy

Policy ARN:
arn:aws:iam::aws:policy/AmazonS3FullAccess

This policy provides full access to Amazon S3 and S3 Object Lambda services.

## Purpose in Project

S3 is used to store output files generated after Kafka data is processed by Spark Streaming.

The project data flow is:

OpenSky API -> Kafka Producer -> Kafka Broker on EC2 -> Spark Streaming on EC2 -> Amazon RDS (Aggregated Data) -> Amazon S3 (Clean Data)


## Data Stored in S3

The S3 bucket can store:

- Processed flight data
- Aggregated flight reports
- Backup files
- Historical data archives

## Required Access

The project used S3 access through IAM permissions.

Required S3 actions include:

```text
s3:ListBucket
s3:GetObject
s3:PutObject
```

In this project, the AWS managed policy `AmazonS3FullAccess` was attached, which provides broader S3 permissions.

## Security Notes

- Do not commit AWS access keys or secret keys to GitHub.
- Keep the S3 bucket private.
- Block public access unless the project specifically requires public files.
- Use IAM roles or IAM users with only the permissions required by the application.
- For production, prefer least-privilege permissions instead of full S3 access.
